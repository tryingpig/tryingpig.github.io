1. 챗봇 vs 에이전트 — 결정적 차이

가장 흔한 오해가 "AI 에이전트 = 좀 더 똑똑한 챗봇"이라는 것이다. 아니다. 둘은 하는 일의 종류가 다르다. 챗봇은 말을 돌려주고, 에이전트는 행동을 돌려준다.

챗봇 (예: 일반 질문답변)AI 에이전트
내가 주는 것질문 ("항공권 어떻게 사?")목표 ("제주 3일 여행 예약해줘")
돌려받는 것설명·답변(글)완료된 결과(실제 예약)
스스로 하는 일없음 — 내가 시키는 한 턴만계획 세우고 여러 단계를 알아서 실행
도구 사용거의 없음검색·앱·API·코드를 직접 씀
비유방법을 알려주는 비서일을 대신 하는 직원

2. 비유: 새로 온 인턴 직원

가장 직관적인 비유는 "새로 입사한 똑똑한 인턴"이다. 상사(=나)가 "이번 주 경쟁사 3곳 가격을 조사해서 표로 정리해 메일 보내줘"라고 하면, 인턴은 이렇게 움직인다:

지시 받기 — "경쟁사 가격 조사 → 표 → 메일"이라는 목표를 이해한다.
스스로 쪼개기 — '어디서 찾지 → 뭘 정리하지 → 어떻게 보내지'로 할 일을 나눈다.
도구 쓰기 — 웹 검색을 하고, 스프레드시트를 열고, 메일 앱을 연다.
막히면 되묻거나 다시 시도 — 정보가 없으면 다른 출처를 찾고, 표가 이상하면 고친다.
보고 — "완료했습니다, 여기 결과입니다"라고 돌려준다.

챗봇은 이 중 "이렇게 하시면 됩니다"라고 방법만 알려주는 선배다. 에이전트는 그 일을 실제로 끝까지 해오는 인턴이다. 이 차이가 왜 산업적으로 중요하냐면 — 사람의 '노동'을 일부 대신하기 때문이다. 그래서 시장이 크고, 그래서 통제(보안)가 필요하다.

3. 에이전트를 이루는 4가지 부품

인턴이 일을 해내려면 머리·판단·손·기억이 필요하듯, AI 에이전트도 네 부품으로 이뤄진다.

① 두뇌 (LLM)

GPT·Claude 같은 대규모 언어모델. "무엇을 할지" 판단하는 추론 엔진. 인턴의 '머리'.

② 계획 (Planning)

큰 목표를 실행 가능한 단계로 쪼갠다. "예약 = 검색 → 비교 → 결제 → 확인" 처럼.

③ 도구 (Tools)

실제로 '손'을 쓰는 부분. 웹 검색, 앱 조작, 코드 실행, 외부 API 호출. 여기서 진짜 '행동'이 나온다.

④ 기억 (Memory)

앞 단계에서 뭘 했는지, 사용자가 뭘 원했는지 기억한다. 이게 없으면 매 단계 초기화돼 일을 못 이어간다.

투자자 관점 힌트: 이 4부품을 보면 "돈이 어디서 벌리는지"가 보인다. 두뇌(①)는 모델 회사(OpenAI·Anthropic), 도구·연결(③)은 인프라·SaaS, 그리고 도구를 쓸 때 지나가는 '통신·데이터 길목'은 보안 회사(예: 지스케일러)의 몫이다. 에이전트가 손(③)을 쓸수록 그 손이 지나가는 길목이 붐빈다 — 이게 앞 시리즈의 핵심이었다.

4. 실제로 한 번 굴러가는 장면

추상적이니 딱 한 번, "제주 여행 3일 예약해줘"라는 목표로 에이전트가 어떻게 도는지 스토리보드로 보자.

단계에이전트가 하는 일쓰는 부품
1목표 해석: "제주, 3일, 예약(항공+숙소)"로 이해두뇌
2계획 수립: 항공 검색 → 숙소 검색 → 일정 조합 → 예약 → 확인 메일계획
3항공 검색 도구 호출 → 가격·시간 비교도구
4숙소 검색 → 위치·가격 필터도구+기억
5충돌 발견(항공 도착이 늦음) → 계획 수정두뇌+계획
6결제·예약 실행 → 확인번호 저장도구+기억
7"완료: 일정·확인번호" 요약 보고두뇌

※ 핵심은 5번 "막히면 스스로 계획을 고쳐 다시 시도"하는 반복 루프다. 챗봇에는 없는 이 자기수정 루프가 에이전트의 본질이다.

5. 자율성 레벨 — 자율주행에 빗대어

"에이전트"라고 다 같은 게 아니다. 자동차 자율주행이 L0~L5로 나뉘듯, 에이전트도 사람이 얼마나 개입하느냐로 단계가 있다. 투자·뉴스에서 "에이전트 도입"이라고 할 때 실제로 몇 단계인지 구분해서 봐야 과장에 안 속는다.

레벨설명비유(운전)
L1사람이 다 하고 AI는 제안만 (자동완성)차선이탈 경고
L2단일 작업을 대신하되 매번 사람이 확인어댑티브 크루즈
L3여러 단계를 스스로, 중요한 순간만 사람에게 물음조건부 자율주행
L4정해진 범위 안에선 거의 완전 자율지정구역 무인택시
L5어떤 목표든 완전 자율 (아직 미래)완전 자율주행
2026년 현재의 현실: 대부분의 상용 '에이전트'는 L2~L3에 있다. 즉 사람이 최종 확인·승인을 하는 '반자율'이 표준이다. "완전 자율 에이전트가 사람을 대체한다"는 말은 아직 마케팅에 가깝다. 투자할 땐 이 눈금으로 과장을 걸러내자.

6. 일상 속 사례 4가지

💻 코딩

"이 버그 고쳐줘" → 코드 읽고, 수정하고, 테스트 돌리고, 안 되면 다시 고침. (개발 보조 에이전트)

📞 고객 상담

단순 안내를 넘어, 주문 조회·환불 처리·배송 변경까지 시스템을 직접 조작.

🔎 리서치

"경쟁사 5곳 비교표 만들어줘" → 검색·수집·요약·표 작성을 한 번에.

🗓️ 개인 비서

일정 조율, 예약, 메일 초안·발송 등 여러 앱을 넘나들며 처리.

공통점: 전부 "여러 단계 + 도구 사용 + 자기수정"이 들어간다. 한 번의 질문답변으로 끝나지 않는다는 게 에이전트의 표식이다.

7. 왜 하필 지금 뜨나

에이전트 개념 자체는 오래됐다. 그런데 2024~2026년에 갑자기 현실이 된 데는 세 가지가 겹쳤다.

  1. 두뇌가 똑똑해졌다. 모델의 추론·계획 능력이 '여러 단계를 스스로 이어갈' 수준으로 올라왔다.
  2. 손을 연결하는 표준이 생겼다. MCP(Model Context Protocol) 같은 규약으로 AI가 외부 도구·데이터에 쉽게 붙게 됐다. (인턴에게 회사 시스템 접근 권한을 준 셈)
  3. 비용이 떨어졌다. 같은 작업을 훨씬 싸게 반복 실행할 수 있게 됐다. 자동화의 경제성이 생겼다.
투자 관점 연결: 2번 "도구 연결 표준(MCP·A2A)"이 열렸다는 건, 그 연결 통로를 검사·통제하는 시장도 함께 열렸다는 뜻이다. 에이전트가 회사 데이터·앱에 손을 뻗을수록, 그 손을 누가 통제하느냐가 새 사업이 된다. → 지스케일러 시리즈로 연결.

8. 한계와 위험 — 그리고 보안 연결

에이전트는 만능이 아니다. '스스로 행동'하기 때문에 오히려 새로운 위험이 생긴다. 이게 곧 투자 기회의 뒷면이다.

위험무슨 일이 생기나
환각(Hallucination)틀린 정보를 사실처럼 확신 → 잘못된 '행동'으로 이어지면 챗봇보다 피해가 큼
폭주·오작동잘못된 계획으로 엉뚱한 작업을 대량 실행 (예: 잘못된 대량 메일)
권한·보안에이전트가 회사 데이터·시스템에 접근 → 유출·오남용 통로가 됨
비용·통제자동 반복 실행이 통제 안 되면 비용 폭증, 감사(누가 뭘 했나) 어려움
🔗 여기서 시리즈로 착지: 위 4가지 위험 중 '권한·보안'이 바로 지스케일러가 파는 시장이다. 수만 개의 에이전트가 각자 임시 신원(NHI)으로 데이터·앱에 손을 뻗을 때, "이 에이전트는 누구고, 무엇에 접근해도 되며, 지금 무슨 통신을 하나"를 검문하는 자리 — 그게 앞 리포트에서 다룬 5개 수익 영역의 뿌리다.

9. FAQ

Q. 에이전트랑 ChatGPT는 다른 건가요?

ChatGPT '앱' 자체는 챗봇이지만, 여기에 도구·계획·기억을 붙여 여러 단계를 스스로 실행하게 만들면 '에이전트'가 됩니다. 즉 같은 모델(두뇌)이라도, 혼자 행동하게 엮으면 에이전트가 됩니다.

Q. 지금 에이전트가 사람 일자리를 다 대체하나요?

아직 대부분 L2~L3(반자율)입니다. 사람의 일부 반복 작업을 덜어주는 단계이지, 판단·책임까지 완전히 넘긴 단계는 아닙니다.

Q. 투자자로서 뭘 봐야 하나요?

①두뇌(모델), ②도구·인프라, ③연결 통로의 보안 — 세 층으로 나눠서 "이 회사가 어느 층에 있나"를 구분하세요. 가치가 어느 층으로 몰리는지가 관건입니다. (이 시리즈는 ③보안 층을 파고들었습니다.)

용어 사전

AI 에이전트목표를 주면 스스로 계획·도구사용·자기수정을 반복해 결과를 만드는 AI.
LLMLarge Language Model. 에이전트의 '두뇌' 역할을 하는 대규모 언어모델(GPT·Claude 등).
도구(Tools)에이전트가 실제 행동에 쓰는 외부 기능 — 검색·앱·API·코드 실행.
MCPModel Context Protocol. AI가 외부 도구·데이터에 붙는 표준 통신 규약.
A2AAgent-to-Agent. 에이전트끼리 직접 주고받는 통신.
NHINon-Human Identity. 사람이 아닌 주체(에이전트·API 키 등)의 신원.
자기수정 루프결과를 점검하고 막히면 계획을 고쳐 다시 시도하는 반복 과정. 에이전트의 본질.
자율성 레벨사람 개입 정도(L1~L5). 2026년 상용은 대체로 L2~L3.
